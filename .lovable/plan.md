Confirmed: **A (compact strip)**, weak-topic rule ON, 1 reset/day.

### Your 2 follow-ups answered

**Q: Daily 1 mission? Toh 10 done = din khatam?**
No — that's bad UX. New model:
- **1 official mission/day** (locked, gives +50 once). Keeps focus + prevents grinding for points.
- After completion, card flips to **"Bonus Round"** mode: same/next chapter, unlimited practice, **no points cap**, but counts toward streak, accuracy, badges. Student can keep going as long as they want — just no extra +50 today.
- Tomorrow (IST 00:00) → fresh mission auto-generated by engine using today's activity.

**Q: Completion screen still looks like "in progress"**
Bug: card shows "Mission complete!" button but still looks like an active CTA — no celebration, no "what next". Will fix as part of this plan.

### Plan

**1. Compact strip on dashboard (replaces big hero)**
- New `<TodaysMissionStrip/>` — single row: `🎯 Solid State • 6/10 • Continue →`
- Tap → opens full card in a `<Sheet/>` (bottom sheet on mobile, dialog on desktop)
- Big hero card kept only inside the AI Planner *Today* tab (where space allows)
- Mobile screen real-estate reclaimed; mission still always visible

**2. Post-completion experience**
- On status → `completed` (first time today):
  - Confetti + toast "+50 JEEnie points 🎉"
  - Card flips to **"Mission Complete ✅ — Bonus Round?"** layout
  - Shows: today's stats (X/10 done, accuracy %, points earned)
  - Two CTAs: **"Bonus practice — same chapter"** (no cap) and **"Switch chapter"** (uses 1 daily reset)
  - Strip variant shows: `✅ Done today • Bonus round →`
- Persist `reward_granted` flag (already in schema) so confetti fires exactly once

**3. Mission-aware practice session**
- CTA URL gets `&mission=1`
- `PracticePage` reads it, mounts a slim top strip: `🎯 Mission: 6/10 — keep going!` (live updates via existing realtime sub)
- On hitting target inside the session → inline "Mission Complete! +50 🎉" banner with *Stop* / *Keep going (bonus)* buttons
- Does NOT change `QUESTIONS_PER_BATCH` — batch size unrelated to mission target

**4. Engine improvements**
- Add **weak-topic rule** before `chapter_practice`: any chapter with accuracy < 50% on last 20 attempts → mission becomes "Revise weak spot — {chapter}" (10 Qs, +50)
- Rule order: cold_start → weak_topic → chapter_practice (active) → next_chapter → fallback
- "Switch chapter" link calls existing `reset_today_mission` RPC, capped at 1/day (DB check on `reset_count` column — needs migration)

**5. AI Planner *Today* tab**
- Already shows the full hero card via shared component — keep as-is (no strip there, plenty of space)
